#  🚀 Ejercicio FizzBuzz: Evolución con TDD y Refactorización
###  Este repositorio documenta mi proceso de aprendizaje aplicando Desarrollo Guiado por Pruebas (TDD) y Refactorización de Código utilizando Java y JUnit 5.
##  📋 Descripción del Proyecto:
###   Objetivo es implementar la lógica de "FizzBuzz":
📌Retornar "Fizz" si el número es múltiplo de 3.

📌Retornar "Buzz" si el número es múltiplo de 5.

📌Retornar "FizzBuzz" si es múltiplo de ambos (ej. el número 15).

📌Retornar el número como cadena de texto si no cumple ninguna regla. 

##  🛠️ Proceso de Desarrollo
###  ✅Fase 1: Pruebas Unitarias Manuales (1 al 7)
En esta etapa, apliqué TDD de forma estricta, creando un método de prueba individual para cada número. Esto me permitió asegurar que las reglas básicas de identidad y múltiplos simples funcionaran correctamente.

##  ✅Fase 2: Refactorización Dinámica (8 al 15)
Al identificar patrones repetitivos, decidí optimizar la suite de pruebas. En lugar de seguir escribiendo métodos manuales, implementé un bucle for que valida automáticamente la serie completa hasta el número 15.

##  ✨Mejoras logradas:
###  🎇Reducción de Código: Logré los mismos resultados con un 80% menos de líneas en el archivo de pruebas.

###  🎇Validación del Caso Especial (15): La lógica refactorizada confirmó que la prioridad de "FizzBuzz" sobre las reglas individuales es correcta.

###  🎇Escalabilidad: El test ahora es capaz de validar rangos mayores sin esfuerzo adicional.
